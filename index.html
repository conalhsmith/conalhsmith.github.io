Robocup

The 2024 Robocop competition follows a Harry Potter theme and resembles a mechatronics version of quidditch. Teams of 3 are required to design and build autonomous robots to compete in a knockout competition at the end of the year. The goal of each round is to collect the largest number of weights, points being doubled if weights are delivered to the robot’s home base. The snitch may also be caught for bonus points. The robots must navigate an arena, littered with obstacles and traps such as fake weights and speedbumps. Collecting dummy weights will result in a penalty. Rounds are short, consisting of only 2 minutes meaning that weight detection, collection, and primary areas for teams to focus on. I was primarily responsbile for concept generation, and mechanical design. I was able to design and build a efficient weight collection system using a paddle and a ramp, testing concluding that it successfully collected weights 90% of the time. This design featured a continuity circuit for real/fake weight differentiation and a simple release gate to drop wreights at base. After concluding the mechanical design I designed the weight collection software, and assisted with navigational software (SLAM).

Helicopter Control Porject

2024

The goal of this project was to program a remote-controlled helicopter so that a "pilot" can execute a sortie consisting of the following tasks: 1. Find a direction for takeoff. 2. Rise to an altitude for stable, maneuverable flight. 3. Rotate around a fixed position in several incremental steps. 4. Land back at base, "parking" in a home position. The main output of the project was a program for a TIVA board that contained an interrupt-driven, real- time kernel to control the lift-off, hover, heading, and landing of a helicopter mounted on a remote rig. Notable modules designed during the project included PWM and PID modules to control both the altitude and yaw based on current values and a desired setpoint. The current altitude was determined using an ADC interrupt handler, converting this input value into an altitude percentage. The yaw was monitored with interrupts reading and converting the binary signals of a quadrature optical angle encoder. Since most of the program was developed before we learned about kernels and their implementation, a major challenge arose when integrating our chosen kernel with the pre-existing code. This emphasized the importance of following clean code principles, which facilitated the integration process. Another challenge was the variation between the helicopter simulator and the actual helicopter rig. A flawlessly working helicopter on the simulator required major tweaks to the PID gains when transferred to the rig. The git repository for this project can be found here: https://github.com/conalhsmith/Helicopter-Rig-Control-Project

Line Following Robot

2023

Created from the ground up, including PCB design and assembly, fabrication and development of mechanical components, and programming. Initially this involved using KiCad to develop electrical schematics and PCB layouts for both the main and line sensing circuits. The goal of the main circuit was to distribute power from the batteries to the MCU, motors, and light sensing circuits. This challenged my electrical design skills, but the result was a greater understanding of circuit design and analysis and two functional circuit boards/PCB’s. The next step was to design the mechanical components of the robot within CAD, whilst optimizing for both speed and style. The programming stage seemed relatively simple, however the robot was unable to change speeds, which I discovered was due to the motor drivers not being connected to the PWM outputs of the MCU. This was a major learning lesson as it how a simple mistake within PCB design can effect the entire system. By performing simple wire bypasses and touching up the code, the robot was able to swiftly maneuver the given course.
